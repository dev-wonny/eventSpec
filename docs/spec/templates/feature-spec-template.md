# Feature Spec Template

신규 기능을 추가할 때 아래 문서 세트를 같은 디렉터리에 생성한다.

## 권장 파일 목록

- `README.md`
- `use-cases.md`
- `business-rules.md`
- `api-spec.md`
- `data-spec.md`
- `test-scenarios.md`
- `open-questions.md`
- `traceability.md`

## README.md

- 기능 목표
- 현재 범위 / 제외 범위
- 문서 구성
- 상태 요약

## use-cases.md

- 액터
- 요구사항 ID 목록
- 기본 시나리오
- 예외 시나리오

## business-rules.md

- 비즈니스 규칙 ID 목록
- 요구사항과 규칙 연결
- 정책 확정이 필요한 항목

## api-spec.md

- 입력 대기 항목
- 엔드포인트 목록
- 엔드포인트 상세 명세
- 오류 응답 정책

## data-spec.md

- 데이터 검토 순서
- 논리 개념과 물리 객체 매핑
- 제약 조건과 인덱스 검토
- 구현 연결 포인트

## test-scenarios.md

- 기능 시나리오
- 예외 시나리오
- 비기능 시나리오
- 테스트 레벨 구분

## open-questions.md

- 미정 정책
- 외부 입력 대기 항목
- 구현 전 의사결정 필요 항목

## traceability.md

- 요구사항과 규칙 연결
- 요구사항과 API 연결
- 요구사항과 데이터 연결
- 요구사항과 테스트 연결
- 문서화 완료 상태
