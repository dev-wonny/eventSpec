# Test Scenarios

이 문서는 출석체크 구현 완료 여부를 판단하는 테스트 기준 초안이다. API와 DDL이 확정되면 각 시나리오를 실제 테스트 코드와 연결한다.

## 기능 시나리오

| ID | 시나리오 | 기대 결과 | 연결 규칙 |
| --- | --- | --- | --- |
| ATT-TEST-001 | 허용 시간 내 정상 출석 요청 | 출석이 성공 저장된다 | ATT-RULE-002, ATT-RULE-003 |
| ATT-TEST-002 | 동일 참여자의 중복 출석 요청 | 정책에 맞는 중복 응답을 반환한다 | ATT-RULE-001, ATT-RULE-005 |
| ATT-TEST-003 | 출석 가능 시간이 아닌 상태에서 요청 | 출석 불가 응답을 반환한다 | ATT-RULE-002 |
| ATT-TEST-004 | 존재하지 않는 이벤트 또는 회차로 요청 | 유효성 오류를 반환한다 | ATT-RULE-004 |
| ATT-TEST-005 | 비활성 또는 마감 상태 대상에 요청 | 출석 불가 응답을 반환한다 | ATT-RULE-003 |
| ATT-TEST-006 | 동일 조건의 동시 요청 2건 이상 발생 | 최종 출석 기록은 한 건만 남는다 | ATT-RULE-001, ATT-RULE-006 |
| ATT-TEST-007 | 출석 결과 조회 요청 | 저장된 최종 상태가 일관되게 반환된다 | ATT-RULE-007 |
| ATT-TEST-008 | DB unique 제약 위반 상황 | 비즈니스 오류로 안전하게 변환된다 | ATT-RULE-001, ATT-RULE-006 |

## 비기능 시나리오

### ATT-TEST-009 감사 가능성

- 최소 생성 시각과 추적 가능한 식별 정보가 남아야 한다.

### ATT-TEST-010 시간대 경계값

- 출석 시작/종료 시각 경계에서 정책대로 처리되어야 한다.

## 테스트 레벨 가이드

- 단위 테스트: 규칙 검증, 상태 판정, 중복 정책
- 통합 테스트: API 요청/응답, DB 제약, 트랜잭션 처리
- 저장소 테스트: QueryDSL 조회 조건, 인덱스/제약 활용

## API/DDL 수신 후 추가할 항목

- 필수 요청 필드별 validation 케이스
- 실제 에러 코드와 메시지 매핑
- 테이블/인덱스 기반 동시성 테스트 방식
