# Data Spec

이 문서는 제공 예정인 최소 DDL을 출석체크 관점으로 해석하는 자리다. 현재는 어떤 항목을 확인해야 하는지와 문서화 형식을 먼저 정의한다.

## 입력 대기 항목

- 최소 DDL 원본
- 테이블 간 관계 설명
- 상태값/코드값 설명

## 데이터 검토 순서

1. 출석 대상 식별에 필요한 테이블을 찾는다.
2. 참여자 식별에 필요한 컬럼을 찾는다.
3. 출석 기록 저장 위치를 찾는다.
4. 중복 방지용 unique key 또는 index를 확인한다.
5. 시간/상태/감사 컬럼을 확인한다.

| ID | 논리 개념 | 물리 객체 | 핵심 컬럼 | 관련 요구사항 | 확인 상태 | 메모 |
| --- | --- | --- | --- | --- | --- | --- |
| ATT-DATA-001 | 출석 대상 식별자 | TBD | TBD | ATT-REQ-001 | DDL 대기 | 이벤트/회차 구조 확인 필요 |

| ATT-DATA-003 | 출석 기록 | TBD | TBD | ATT-REQ-002, ATT-REQ-005 | DDL 대기 | 저장 테이블 확인 필요 |

## DDL 반영 체크리스트

- 출석 단위가 이벤트인지 회차인지 날짜인지
- soft delete 또는 상태 컬럼 존재 여부
- 시간대 처리 기준
- 고유성 제약이 DB 레벨에 있는지
- 후속 기능이 참조할 FK 또는 조인 경로가 있는지

## 구현 연결 포인트

- Entity 후보 정의
- JPA 연관관계/비연관 조회 전략
- QueryDSL 조회가 필요한 화면 또는 API
- 인덱스와 동시성 위험 구간
