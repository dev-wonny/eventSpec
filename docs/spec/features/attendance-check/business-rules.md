# Business Rules

이 문서는 출석체크 기능에서 구현 로직으로 옮겨질 정책을 정의한다. 상세 값은 DDL/API/운영 정책 입력 후 보완한다.

## 규칙 초안

### ATT-RULE-001 출석 고유성

- 동일 참여자는 동일 출석 기준 단위에 대해 한 번만 유효한 출석 기록을 가질 수 있다.
- 고유성 기준 컬럼 조합은 DDL 확인 후 확정한다.

### ATT-RULE-002 출석 가능 시간

- 출석은 허용된 시간 구간 내에서만 처리된다.
- 시간 구간이 없거나 상시 허용인지 여부는 정책 확정이 필요하다.

### ATT-RULE-003 출석 가능 상태

- 출석 대상은 출석 가능한 상태여야 한다.
- 예: 공개 여부, 진행 상태, 삭제 여부, 마감 여부

### ATT-RULE-004 입력값 검증

- 서버는 필수 식별값 누락, 형식 오류, 존재하지 않는 대상을 거절해야 한다.
- 참여자 식별 방식은 인증 정보 또는 요청 필드 중 어디에서 오는지 확정이 필요하다.

### ATT-RULE-005 중복 요청 처리 정책

- 같은 요청이 짧은 시간 내 반복될 수 있으므로 멱등성 정책이 필요하다.
- 비즈니스 응답 기준과 DB 제약 조건 기준을 함께 정의한다.

### ATT-RULE-006 동시성 제어

- 동시에 여러 요청이 들어와도 최종 저장 결과는 한 번만 반영되어야 한다.
- 애플리케이션 레벨 검증과 DB 제약 중 어디를 주 방어선으로 삼을지 결정해야 한다.

### ATT-RULE-007 이력 및 감사

- 최소한 생성 시각, 수정 시각, 처리 주체 등 감사에 필요한 정보를 검토한다.
- 취소/정정 기능이 없다면 현재는 생성 이력 중심으로 단순화할 수 있다.

## 규칙과 요구사항 연결

- `ATT-REQ-001`: ATT-RULE-002, ATT-RULE-003, ATT-RULE-004
- `ATT-REQ-002`: ATT-RULE-002, ATT-RULE-003, ATT-RULE-004
- `ATT-REQ-003`: ATT-RULE-001, ATT-RULE-005, ATT-RULE-006
- `ATT-REQ-004`: ATT-RULE-004, ATT-RULE-007
- `ATT-REQ-005`: ATT-RULE-001, ATT-RULE-007

## DDL 수신 후 바로 확인할 항목

- Unique constraint 후보
- 상태값 컬럼 존재 여부
- 시간 관련 컬럼 존재 여부
- 감사 컬럼 존재 여부
