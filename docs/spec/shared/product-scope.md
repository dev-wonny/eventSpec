# Product Scope

## 서비스 목표

- 이벤트 서비스를 점진적으로 확장 가능한 구조로 만든다.
- 현재 1차 목표는 출석체크 기능을 안정적으로 출시하는 것이다.
- 이후 2차 목표로 랜덤 게임 기능을 추가한다.

## 단계별 범위

### Phase 1. 출석체크

- 이벤트 참여자의 출석 처리
- 출석 가능 여부 판별
- 중복 출석 방지
- 출석 결과 조회 또는 확인

### Phase 2. 랜덤 게임

- 출석체크 이후 참여 가능한 추가 이벤트성 기능
- 세부 룰은 추후 정의

## 현재 In Scope

- 출석체크 기능 spec 정의
- 출석체크용 API 계약 정의
- 제공 예정 DDL과의 데이터 매핑 정의
- 테스트 시나리오 정리

## 현재 Out Of Scope

- 랜덤 게임 상세 규칙
- 리워드/포인트 정책
- 운영자 백오피스 상세 화면
- 알림, 통계, 외부 연동

## 완료 기준

- 출석체크 요구사항, 규칙, API, 데이터, 테스트 문서가 서로 연결된다.
- 제공 예정 DDL과 최소 API 목록이 문서에 반영된다.
- 구현 전에 미정 항목이 `open-questions.md`로 분리된다.
