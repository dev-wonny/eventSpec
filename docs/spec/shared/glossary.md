# Glossary

아래 용어는 현재 spec 초안 기준이며, 실제 DDL/API 명칭이 오면 정식 용어로 보정한다.

## 공통 용어

- `이벤트`: 사용자가 참여하는 상위 행사 또는 캠페인 단위
- `출석체크`: 특정 이벤트 또는 회차에 대해 참여 여부를 기록하는 행위
- `참여자`: 출석체크를 수행하는 사용자
- `출석 요청`: 참여자가 서버에 보내는 출석 처리 요청
- `출석 기록`: 서버가 저장한 최종 출석 결과
- `출석 가능 시간`: 출석이 허용되는 시간 구간
- `중복 출석`: 동일 참여자가 동일 기준 단위에 대해 여러 번 출석을 시도하는 상황
- `운영자`: 이벤트 상태 확인 또는 관리가 필요한 내부 사용자

## 용어 확정이 필요한 항목

- 이벤트와 회차의 관계
- 참여자 식별 키
- 출석 상태 코드 종류
- 운영자 권한 범위
