# Glossary

아래 용어는 현재까지 공유된 DDL, 도메인 설명, 확정된 외부 API 계약을 기준으로 정리한다.

## 공통 용어

- `이벤트 (event)`: 출석체크나 랜덤 보상 기능을 담는 상위 단위
- `이벤트 회차 (event_round)`: 이벤트 하위의 참여 단위. 출석체크에서는 일자성 회차로 해석하며, 예를 들어 2월 출석체크 이벤트는 같은 `event_id` 아래 28개의 회차를 가질 수 있다.
- `참여 가능 대상자 (event_applicant)`: 해당 이벤트에 참여할 수 있는 사용자 대상자 풀. 참여 조건이나 대상자를 사전에 계산해 적재해두는 eligibility 기준 테이블이며, 기본 식별은 `(event_id, member_id)`다. 이벤트 기간 조건을 만족하는 상태에서 `event_applicant`에 있으면 추가 참여 조건 조회 없이 참여 가능자로 본다. `round_id`는 특정 회차에만 참여 가능하도록 제한할 때 선택적으로 사용한다.
- `참여 엔트리 (event_entry)`: 실제 참여 기록을 append-only로 누적하는 레코드. 출석 성공 이력과 향후 랜덤 리워드 참여 이력이 여기에 쌓이며, 출석 이벤트에서는 일별 출석 여부 판정을 위해 `event_id`, `round_id`를 가진다.
- `출석 요청`: 참여자가 특정 이벤트/회차에 대해 출석을 시도하는 요청
- `출석 기록`: 현재 합의 기준에서는 `event_entry`에 저장되는 일별 출석 성공 이력
- `출석 가능 시간`: 이벤트와 회차의 시간 조건을 모두 만족해야 하는 출석 허용 구간
- `중복 출석`: 동일 참여자가 동일 `event_id + round_id + member_id` 기준으로 여러 번 출석을 시도하는 상황
- `공급사 식별자 (supplier_id)`: 이벤트에 연결되는 외부 값참조 식별자. 현재는 위드 DB 기준 값을 사용하고, 이후 버터가 쇼핑몰 외주 개발을 완료하면 위드 데이터를 버터 DB로 마이그레이션해 사용할 예정이다. 현재 용도는 돌쇠네 자체 서비스 범위다.
- `운영자`: 이벤트 상태 확인 또는 관리가 필요한 내부 사용자

## 미래 확장 용어

- `경품 (prize)`: 재사용 가능한 보상 마스터. 출석체크에서는 주로 `POINT` 유형을 사용한다.
- `회차별 경품 정책 (event_round_prize)`: 특정 회차에 연결된 경품 정책. 출석체크에서는 회차당 `0..1`개를 두고, 랜덤 리워드에서는 같은 회차에 여러 정책을 연결할 수 있다.
- `불변 경품 마스터`: 운영 합의상 한 번 세팅이 완료된 `prize`는 수정하지 않고, 변경이 필요하면 새 `prize`를 생성해서 연결을 바꾸는 방식
- `보상 결과 이력 (event_win)`: 실제로 어떤 보상이 지급되었는지와 외부 보상 API 요청 성공 이력을 남기는 레코드. 출석체크에서는 보상이 실제 지급된 경우에만 이력이 남는다.
- `외부 보상 API`: point 적립 등 실제 보상 지급을 처리하는 외부 시스템 호출

## 용어 확정이 필요한 항목

- 참여자 식별 키의 입력 방식
- 운영자 권한 범위
