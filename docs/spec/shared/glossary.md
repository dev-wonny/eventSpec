# Glossary

아래 용어는 현재까지 공유된 DDL과 도메인 설명을 기준으로 정리한다. API 목록이 오면 요청/응답 용어를 추가로 맞춘다.

## 공통 용어

- `이벤트 (event)`: 출석체크나 랜덤 보상 기능을 담는 상위 단위
- `이벤트 회차 (event_round)`: 이벤트 하위의 참여 단위. 출석체크에서는 일자성 회차로 해석하며, 예를 들어 2월 출석체크 이벤트는 같은 `event_id` 아래 28개의 회차를 가질 수 있다.
- `참여자 앵커 (event_applicant)`: 사용자가 해당 이벤트 참여자임을 한 번 확인한 뒤 재사용하는 기준 레코드
- `참여 엔트리 (event_entry)`: 출석 성공 이력과 향후 랜덤 리워드 참여 이력을 append-only로 누적하는 참여 레코드
- `출석 요청`: 참여자가 특정 이벤트/회차에 대해 출석을 시도하는 요청
- `출석 기록`: 현재 합의 기준에서는 `event_entry`에 저장되는 일별 출석 성공 이력
- `출석 가능 시간`: 이벤트와 회차의 시간 조건을 모두 만족해야 하는 출석 허용 구간
- `중복 출석`: 동일 참여자가 동일 회차 또는 동일 출석일 기준으로 여러 번 출석을 시도하는 상황
- `운영자`: 이벤트 상태 확인 또는 관리가 필요한 내부 사용자

## 미래 확장 용어

- `경품 (prize)`: 재사용 가능한 보상 마스터. 출석체크에서는 주로 `POINT` 유형을 사용한다.
- `회차별 경품 정책 (event_round_prize)`: 특정 회차에 연결된 경품 정책. 출석체크에서는 날짜별 회차에 point 보상을 연결하는 용도로 주로 사용한다.
- `불변 경품 마스터`: 운영 합의상 한 번 세팅이 완료된 `prize`는 수정하지 않고, 변경이 필요하면 새 `prize`를 생성해서 연결을 바꾸는 방식
- `보상 결과 이력 (event_win)`: 실제로 어떤 보상이 지급되었는지와 외부 보상 API 요청 성공 이력을 남기는 레코드. 출석체크에서는 point 지급 성공 이력이 여기에 남는다.
- `외부 보상 API`: point 적립 등 실제 보상 지급을 처리하는 외부 시스템 호출

## 용어 확정이 필요한 항목

- `event_applicant.round_id`의 업무 의미
- `event_entry`와 `event_round`의 최종 매핑 방식
- 참여자 식별 키의 입력 방식
- 운영자 권한 범위
